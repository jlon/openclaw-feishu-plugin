# 群聊协作模式

## 1. 模式总览

当前插件支持 4 种群聊协作模式。

### `default`

适用场景：
- 无 `@`
- 普通群任务入口

行为：
- 只有 `main` 处理

### `direct_reply`

适用场景：
- 同时点名多个 Agent，但只是轻量直答
- 例如：打招呼、一个字描述、各自介绍

行为：
- 每个被点名 Agent 只代表自己说一句
- 不创建协作任务
- 不进入 handoff

### `peer_collab`

适用场景：
- 同时点名多个内部专业 Agent
- 明确要求“一起看、互相补充、继续讨论、协作定位”

行为：
- 创建协作任务
- 初始并行判断
- 选出当前 owner
- 允许结构化 handoff
- 支持受控多轮协作

### `coordinate`

适用场景：
- 明确要求 `main` 安排、拉通、汇总

行为：
- 只有 `main` 处理原始群消息
- 专业 Agent 通过内部协作参与
- 最终由 `main` 汇总

## 2. 关键原则

1. 可见 `@` 是展示层
2. 稳定控制面是内部协作协议
3. 外部 bot 不进入内部 handoff 主链
4. 共同点名直答和复杂协作必须严格区分

## 3. 典型例子

### 轻量共同点名直答

```text
@Flink-SRE @Starrocks-SRE 一个字描述下 john
```

预期：
- 两个 Agent 各回自己的一句
- 不创建协作任务

### 专业 Agent 协作

```text
@Flink-SRE @Starrocks-SRE 你俩一起看下这条链路，先各自说判断，再互相补充
```

预期：
- 进入 `peer_collab`
- 允许多轮协作

### main 编排

```text
@首席大管家 @Flink-SRE @Starrocks-SRE 帮我安排并汇总这次排查
```

预期：
- 进入 `coordinate`
- 只有 `main` 接原始群消息
- 由 `main` 编排并最终汇总
